fn dispatch_new_events(&mut self) -> Result<(), WinitInputError>
[src]

Processes new events of the underlying event loop to drive the set InputHandler.

You need to periodically call this function to keep the underlying event loop and Window active. Otherwise the window may no respond to user interaction and no input events will be received by a set InputHandler.

Returns an error if the Window the window has been closed. Calling dispatch_new_events again after the Window has been closed is considered an application error and unspecified baviour may occur.

The linked WinitGraphicsBackend will error with a lost Context and should not be used anymore as well.
